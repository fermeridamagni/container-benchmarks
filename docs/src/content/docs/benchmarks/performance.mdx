## Overview

Performance is critical when evaluating a container runtime. We have benchmarked OrbStack and Apple Containers across different images using robust `criterion` tests in Rust.

### Container Startup Time

The startup time measures how fast a container can boot, execute a command (`echo booted`), and terminate. Apple Containers launch a dedicated micro-VM per container, which provides great isolation but can have a slight overhead compared to OrbStack.

### Memory Usage (Overhead)

Apple Containers consume a baseline of RAM for their dedicated VM instance per container. OrbStack uses a shared Linux VM, which means launching multiple containers consumes significantly less memory overall.
